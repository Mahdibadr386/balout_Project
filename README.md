# 🛍️ Laravel E-Commerce API

یک **API فروشگاهی** قدرتمند و مقیاس‌پذیر که با **Laravel** توسعه داده شده و بر پایه‌ی معماری تمیز و اصول SOLID بنا شده است. این پروژه با هدف داشتن کدی خوانا، قابل نگهداری و قابل تست طراحی شده و از الگوهای طراحی استاندارد صنعت بهره می‌برد.

---

## 📌 درباره پروژه

این پروژه یک **RESTful API** کامل برای یک فروشگاه اینترنتی است که شامل مدیریت محصولات، دسته‌بندی‌ها، سبد خرید، سفارش‌ها، کاربران و پرداخت می‌شود. تمرکز اصلی پروژه بر **جداسازی مسئولیت‌ها (Separation of Concerns)** و نوشتن کدی تمیز و قابل توسعه است.

---

## 🏗️ معماری پروژه

پروژه از الگوهای طراحی زیر پیروی می‌کند:

- **Repository Pattern**  
  جداسازی منطق دسترسی به داده از منطق تجاری (Business Logic)، که باعث افزایش تست‌پذیری و کاهش وابستگی مستقیم به Eloquent در لایه‌های بالاتر می‌شود.

- **Single Action Controller**  
  هر کنترلر تنها مسئول انجام یک عملیات مشخص است (`__invoke`)، که خوانایی کد را بالا برده و اصل تک‌مسئولیتی (Single Responsibility Principle) را رعایت می‌کند.

- **Service Layer**  
  منطق تجاری پیچیده از کنترلرها جدا شده و در لایه‌ی Service پیاده‌سازی می‌شود.

- **DTO (Data Transfer Object)**  
  برای انتقال امن و ساختاریافته‌ی داده بین لایه‌های مختلف برنامه.

- **Form Request Validation**  
  اعتبارسنجی ورودی‌ها به‌صورت مجزا از کنترلر، با استفاده از کلاس‌های اختصاصی `FormRequest`.

- **API Resource**  
  شکل‌دهی و استانداردسازی خروجی JSON با استفاده از `JsonResource`.
